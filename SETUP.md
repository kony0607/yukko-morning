## 🏃 実際の業務フロー

### **朝：スタッフがPC で入力 [8:00]**

1. **スタッフ入力タブ** で紙注文表から登録
   - ロッカー番号 `10001`
   - 商品 `和食`
   - 入力 → **「この行を追加」** ボタン
   - 自動でサーバー保存 ✓

2. **受け取り受付を開始** クリック
   - キッチン側に受付順リストが表示される
   - スマホ側で QR/手入力 準備完了

---

### **営業中：顧客がスマホで受け取り [11:30]**

1. **受け取り確認タブ** が開かれている
   - ロッカー番号 `10001` のバーコード読み込み
   - または手入力

2. 自動で以下が発生：
   - ✅ スタッフPC画面の「001」が「受取済み」に変わる
   - ✅ 顧客スマホに「和食弁当 / ご飯200g」表示
   - ✅ ピックアップログに記録
   - ✅ サーバーのDBに保存

## 🛠️ トラブルシューティング

### **Q: ポート 3000 が既に使われている**

```bash
# ポート確認
lsof -i :3000

# 別ポートで起動（server.jsを編集）
# const PORT = 3001;  # に変更
npm start
```

### **Q: npm install でエラー**

```bash
# キャッシュクリア
npm cache clean --force

# 再試行
npm install
```

### **Q: データベースが壊れた**

```bash
# DB削除してリセット
rm orders.db

# 再起動（自動再作成）
npm start
```

### **Q: localhost でなく別PCからアクセスしたい**

```bash
# PC のIPを確認
ifconfig | grep "inet "
# 例: 192.168.1.100

# 別PCのブラウザで
http://192.168.1.100:3000
```
